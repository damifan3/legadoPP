# 定时任务帮助

定时任务可以在后台定时执行指定的 JavaScript 脚本，可用于实现自动签到、自动刷新特定书籍目录、自动购买等功能。

* **基本设置**
  * **名称**：任务的名称，用于显示和日志区分。
  * **Cron 表达式**：定义任务的执行频率。采用标准的 5 字段格式：`分 时 日 月 周`。
    * 示例：`*/30 * * * *` (每小时的00分和30分执行一次)
    * 示例：`0 * * * *` (每小时的第0分钟执行)
    * 支持通配符 `*` `?`，范围 `1-5`，步长 `*/5` `1-10/2`，列表 `1,3,5`。周 0 和 7 均表示周日。
  * **并发率**：格式为 "次数/毫秒"，例如 `10/1000` 表示 1秒内最多执行 10 次请求。
  * **脚本**：需要执行的 JS 代码，可直接书写，也可使用 `@js:` 或 `<js>...</js>` 包装；也可以直接返回一个字典。
  * **请求头**：HTTP 请求头配置 (JSON 格式)。
  * **JsLib**：在脚本执行前会预加载的 JS 库代码。
  * **登录配置**：如果需要处理登录，可配置登录URL、登录UI和检查JS，用法同书源登录。

* **JS 执行与返回值**
  * 脚本中可以使用网络请求方法，也可引用 `source` 对象获取当前配置。
  * **返回值协议**：如果脚本返回特定格式的 JSON 对象或数组，将触发应用的内置动作。目前支持以下两种动作：`refreshToc` (刷新目录) 和 `notify` (发送通知)。

* **动作: 刷新目录 (refreshToc)**
  用于刷新指定书籍的目录，并在发现新章节时可选地发送通知、缓存或自动购买。

  例子：
  ```js
  (function(){
    var bookUrl='https://example.cn/novel/info?novelId=example,{"method":"GET","headers":{"authorization":"example"}}';
    return{
      "type": "refreshToc",
      "bookUrl": bookUrl ,// 必填：要刷新的书籍URL
      "notify": {              // 可选：通知配置
          "enable": true,      // 发现新章节时是否发送系统通知 (默认 false)
          "minCount": 0,       // 触发通知的最小更新章节数 (默认 1)
          "title": "{book}更新了",  // 标题模板
          "content": "新增了{newCount}章，最新：{chapter}\n自动购买成功:{purchaseCount}章" // 内容模板
      },
      "cache": {               // 可选：缓存配置
          "enable": true       // 是否自动缓存新增的章节 (默认 false)
      },
      "purchase": {            // 可选：自动购买配置
          "enable": true,      // 是否自动购买新增的 VIP 章节 (默认 false)
          "maxCount": 2      // 单次最大购买数量限制，防误购 (默认 10)
      }
  };
  })()
  ```
  *说明：模板变量支持：`{book}`(书名)、`{author}`(作者)、`{newCount}`(新增章节数)、`{chapter}`(最新章节标题)、`{time}`(当前时间)、`{purchaseCount}`(购买成功章节数)、`{purchaseFailCount}`(购买失败章节数)。自动购买仅对本次刷新检测到的新增且未购买的 VIP 章节生效。*

* **动作: 自定义通知 (notify)**
  用于在脚本执行后发送自定义内容的系统通知。

  例子：
  ```json
  {
      "type": "notify",
      "title": "签到成功",       // 通知标题
      "content": "获得了 10 个金币", // 通知内容
      "level": "high"            // 通知级别: high/error/fail/low/default
  }
  ```
  *说明：可以在 JS 脚本逻辑中根据执行结果动态组装并返回此对象。返回数组可同时触发多个动作。*

* **注意事项**
  * Android 系统对后台服务的限制较严，为了保证定时任务准时执行，请允许应用后台运行并将其加入省电策略白名单。
  * 每次任务执行的详细结果、耗时和报错信息，可以在每个任务弹出的菜单中选择"日志"页面进行查看。
